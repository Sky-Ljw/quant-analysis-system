# 量化分析系统使用说明

## 系统概述

本系统是一个基于Python的量化分析工具，用于监测主流加密货币和A股股票的涨跌情况。系统具有以下功能：

- 实时监测加密货币（如BTC、ETH等）的价格和涨跌幅
- 实时监测A股股票的价格和涨跌幅
- 当涨跌幅超过设定阈值时发出警报
- 可配置的监测间隔和警报阈值
- 预留了数据可视化功能接口

## 系统架构

系统由以下核心组件组成：

- **QuantAnalysisSystem类**：核心类，负责数据获取、分析和监测
- **get_crypto_data()**：从Binance API获取加密货币数据
- **get_stock_data()**：从Tushare API获取A股数据
- **monitor_market()**：持续监测市场并输出结果
- **calculate_change()**：计算涨跌幅
- **visualize_data()**：数据可视化功能（预留）

## 安装步骤

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 获取API Token

- **加密货币数据**：使用Binance公开API，无需注册
- **A股数据**：需要注册Tushare账号并获取API Token
  1. 访问 https://tushare.pro/register 注册账号
  2. 登录后在个人中心获取Token
  3. 将Token填入config.json文件中的`tushare_token`字段

### 3. 配置系统

运行以下命令创建默认配置文件：

```bash
python quant_analysis_system.py
```

然后编辑config.json文件，根据需要修改配置：

- `crypto_symbols`：要监测的加密货币列表
- `stock_symbols`：要监测的A股股票列表
- `refresh_interval`：数据刷新间隔（秒）
- `alert_threshold`：涨跌幅警报阈值（百分比）

## 使用方法

### 启动监测

```bash
python quant_analysis_system.py monitor
```

系统会持续监测市场，并在终端输出实时数据。当涨跌幅超过设定阈值时，会发出警报。

### 配置示例

```json
{
    "crypto_symbols": ["BTC", "ETH", "USDT", "BNB", "ADA"],
    "stock_symbols": ["600000.SH", "600519.SH", "000001.SZ", "000858.SZ", "000333.SZ"],
    "crypto_api_url": "https://api.binance.com/api/v3/ticker/price",
    "stock_api_url": "https://api.tushare.pro",
    "tushare_token": "your_tushare_token_here",
    "refresh_interval": 60,
    "alert_threshold": 5.0
}
```

## 注意事项

1. **网络连接**：系统需要稳定的网络连接以获取实时数据
2. **API限制**：
   - Binance API有访问频率限制，请合理设置refresh_interval
   - Tushare API免费用户有调用次数限制
3. **数据延迟**：由于API响应时间和网络延迟，数据可能存在一定的滞后
4. **环境变量**：在某些环境中，可能需要设置代理以访问外部API

## 故障排除

### 常见问题

1. **无法获取加密货币数据**
   - 检查网络连接
   - 检查Binance API是否可访问
   - 可能需要设置代理

2. **无法获取A股数据**
   - 检查Tushare Token是否正确
   - 检查Tushare API调用次数是否超限
   - 检查网络连接

3. **系统运行缓慢**
   - 增加refresh_interval值
   - 减少监测的币种和股票数量

## 扩展功能

系统预留了以下扩展功能的接口：

- **历史数据存储**：可添加数据库存储功能，保存历史数据
- **技术分析指标**：可实现MACD、KDJ等技术分析指标
- **交易信号生成**：可根据涨跌幅和技术指标生成交易信号
- **数据可视化**：可使用matplotlib实现数据可视化
- **Web界面**：可开发Web界面，提供更直观的监测体验

## 技术支持

如果遇到问题，请检查：

2. API Token是否正确配置
3. 网络连接是否正常
4. 配置文件是否正确设置
